# v1.0.0 — Signed release APK + version hiển thị — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Đóng bản chính thức Phase 1: build CLI ra APK **release đã ký** bằng keystore riêng (không Android Studio), và hiện **"Phiên bản X.Y.Z"** trong Cài đặt — đọc tự động từ một nguồn (package.json). KHÔNG thêm tính năng mới.

**Architecture:**
- **Single source of truth cho version = `package.json`.** `android/app/build.gradle` đọc `versionName` từ `package.json` (Groovy JsonSlurper); Vite inject `__APP_VERSION__` từ `package.json` làm fallback cho web/dev. Không hardcode chuỗi version trong UI.
- **Signing:** keystore nằm NGOÀI repo (`~/keystores/gu-library/`), credential trong `android/keystore.properties` (gitignored). `build.gradle` nạp file đó (nếu có) → `signingConfigs.release` → `buildTypes.release`. Vắng file thì build debug vẫn chạy bình thường (CI-safe).
- **Version UI:** `@capacitor/app` `App.getInfo().version` (native versionName, = package.json sau khi build) hiển thị ở cuối Settings; lỗi (web/dev) → fallback `__APP_VERSION__`.

**Tech Stack:** Capacitor 8 (`@capacitor/app` 8.1.0 đã cài), Ionic React, Vite, Android Gradle (JDK 21 / SDK 36, cmdline-tools, KHÔNG Android Studio), `keytool` (đi kèm JDK), `apksigner`/Gradle signing.

**Bối cảnh nền (đã xác định):**
- `android/` ĐƯỢC track trong git (`android/app/build.gradle` tracked) → sửa build.gradle persist qua `cap sync`.
- build.gradle hiện: `versionCode 1`, `versionName "1.0"` (hardcoded), `applicationId "com.gulibrary.app"`. `buildTypes.release` chưa có `signingConfig`.
- `android/.gitignore` có sẵn dòng `#*.jks` / `#*.keystore` (đang comment) — sẽ bật + thêm `keystore.properties`.
- Device thật: Z Flip 4 (SM-F721B, serial `R5CT844VRCN`).
- `package.json` version hiện `0.10.0` → bump `1.0.0`.

**⚠ Cảnh báo keystore (phải nêu lại cho user trong output cuối):** mất file keystore HOẶC quên mật khẩu = **không update được app đã cài** (phải gỡ rồi cài lại từ đầu, mất dữ liệu app). Phải backup: (1) file `.jks`, (2) `keystore.properties` (chứa 2 mật khẩu + alias) HOẶC tự ghi lại `keyAlias` + `storePassword` + `keyPassword`.

## Task 1: Single-source version (package.json → build.gradle + Vite)

**Files:**
- Modify: `package.json` (version)
- Modify: `android/app/build.gradle`
- Modify: `vite.config.ts`

- [ ] **Step 1: Bump package.json**

`package.json`: `"version": "0.10.0"` → `"version": "1.0.0"`.

- [ ] **Step 2: build.gradle đọc versionName từ package.json + bump versionCode**

Trong `android/app/build.gradle`, NGAY SAU dòng `apply plugin: 'com.android.application'` (dòng 1), thêm:

```groovy
import groovy.json.JsonSlurper

def appVersion = new JsonSlurper().parseText(rootProject.file("../package.json").getText("UTF-8")).version
```

Rồi trong khối `defaultConfig`, đổi hai dòng version:

```groovy
        versionCode 2
        versionName appVersion
```

(versionCode là integer Android, không hiển thị; bump lên 2 cho bản release chính thức đầu tiên — các bản sau tăng tiếp tay. versionName lấy từ package.json = "1.0.0".)

- [ ] **Step 3: Vite inject __APP_VERSION__ (fallback web/dev)**

Sửa `vite.config.ts` thành:

```ts
/// <reference types="vitest" />
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';
import pkg from './package.json';

export default defineConfig({
  plugins: [react()],
  define: {
    __APP_VERSION__: JSON.stringify(pkg.version),
  },
  test: {
    globals: true,
    environment: 'jsdom',
    setupFiles: './src/setupTests.ts',
  },
});
```

Nếu `tsc` báo lỗi import JSON, đảm bảo `tsconfig` cho phép (`resolveJsonModule` thường đã bật với Vite). Nếu chưa, dùng cách đọc không-import:
```ts
import { readFileSync } from 'node:fs';
const pkg = JSON.parse(readFileSync(new URL('./package.json', import.meta.url), 'utf-8'));
```
(Ưu tiên cách `import pkg` trước; chỉ đổi nếu lỗi.)

- [ ] **Step 4: Khai báo type cho __APP_VERSION__**

Nếu `src/vite-env.d.ts` chưa tồn tại, tạo nó:
```ts
/// <reference types="vite/client" />

declare const __APP_VERSION__: string;
```
Nếu đã tồn tại, chỉ thêm dòng `declare const __APP_VERSION__: string;` vào cuối.

## Task 3: Keystore + signing config (ONE-TIME, xử lý secret cẩn thận)

> **Thực thi INLINE bởi controller, KHÔNG giao subagent** — bước này sinh credential nhạy cảm; mật khẩu KHÔNG được lọt vào chat/transcript/subagent. Sinh mật khẩu ngẫu nhiên mạnh tại máy, ghi thẳng vào `android/keystore.properties` (gitignored). KHÔNG in mật khẩu ra output.

**Files:**
- Create: `~/keystores/gu-library/gu-library-release.jks`
- Create: `android/keystore.properties`
- Modify: `android/.gitignore`, `android/app/build.gradle`

- [ ] **Step 1: Bật gitignore TRƯỚC khi tạo bất kỳ credential nào**

Trong `android/.gitignore`, đổi hai dòng đang comment thành active và thêm keystore.properties:
```
# Keystore files
*.jks
*.keystore
keystore.properties
```
Verify: `git check-ignore android/keystore.properties android/app/foo.jks` phải in ra hai path (nghĩa là đã ignore).

- [ ] **Step 2: Tạo thư mục keystore ngoài repo + sinh mật khẩu mạnh (không in ra)**

```bash
mkdir -p ~/keystores/gu-library
STOREPASS="$(openssl rand -base64 24)"
KEYPASS="$STOREPASS"   # dùng chung 1 mật khẩu cho store + key (đơn giản, vẫn an toàn cho app cá nhân)
```
(Giữ biến trong cùng một shell session cho Step 3-4. KHÔNG `echo` mật khẩu.)

- [ ] **Step 3: Tạo keystore bằng keytool (JDK 21)**

```bash
source ~/.bash_profile
keytool -genkeypair -v \
  -keystore ~/keystores/gu-library/gu-library-release.jks \
  -alias gu-library \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass "$STOREPASS" -keypass "$KEYPASS" \
  -dname "CN=Gu Library, OU=Personal, O=Gu, L=, ST=, C=VN"
```
Verify file tồn tại: `ls -la ~/keystores/gu-library/gu-library-release.jks`.

- [ ] **Step 4: Ghi `android/keystore.properties` (gitignored)**

```bash
cat > /Users/lavopavden/Dev/projects/Gu-Library/android/keystore.properties <<EOF
storeFile=${HOME}/keystores/gu-library/gu-library-release.jks
storePassword=${STOREPASS}
keyAlias=gu-library
keyPassword=${KEYPASS}
EOF
```
Verify KHÔNG bị track: `git status --porcelain android/keystore.properties` phải RỖNG (ignored).

- [ ] **Step 5: build.gradle nạp keystore.properties → signingConfigs.release**

Trong `android/app/build.gradle`:

(a) Sau dòng `def appVersion = ...` (Task 1 Step 2), thêm:
```groovy
def keystorePropsFile = rootProject.file("keystore.properties")
def keystoreProps = new Properties()
if (keystorePropsFile.exists()) {
    keystoreProps.load(new FileInputStream(keystorePropsFile))
}
```

(b) Trong khối `android { ... }`, thêm `signingConfigs` NGAY TRƯỚC `defaultConfig`:
```groovy
    signingConfigs {
        release {
            if (keystorePropsFile.exists()) {
                storeFile file(keystoreProps['storeFile'])
                storePassword keystoreProps['storePassword']
                keyAlias keystoreProps['keyAlias']
                keyPassword keystoreProps['keyPassword']
            }
        }
    }
```

(c) Trong `buildTypes.release`, thêm dòng signingConfig (có guard):
```groovy
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
            if (keystorePropsFile.exists()) {
                signingConfig signingConfigs.release
            }
        }
```
